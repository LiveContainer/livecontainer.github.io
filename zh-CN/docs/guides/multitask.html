多任务

LiveContainer 可以在多任务模式下运行应用,因此您可以同时启动多个App/容器并快速在它们之间切换。

要求​

  1. iOS/iPadOS 16.0+。
  2. 保留应用扩展安装 LiveContainer(在 SideStore/AltStore 安装流程中保留 Keep App Extensions)。

如果缺少扩展,多任务启动可能会失败,并出现 LiveProcess extension not found 错误。

开始多任务​

  1. 在应用设置中将目标应用转换为共享App(如果尚未共享)。
  2. 长按应用卡片。
  3. 点击 多任务。

您还可以在设置中启用 默认以多任务模式启动,然后主 运行 按钮会直接在多任务模式下启动该应用。

多任务模式​

（在iPad OS 16.1+中可用）

在 设置 -> 多任务 中,当前版本提供两种模式:

  1. 虚拟窗口: 使用应用内浮动窗口(停靠栏、PiP、最大化/最小化行为)。
  2. 原生窗口: 在系统原生窗口中打开每个应用。(需要 iPadOS 16.1+)

多任务设置​

设置 -> 多任务 中可用的选项:

  • 多任务模式: 在 虚拟窗口 和 原生窗口 之间选择。
  • 默认以多任务模式启动: 点击 启动 时默认在多任务模式下启动应用。

当 多任务模式 设置为 虚拟窗口 时,会出现以下选项:

  • 默认最大化多任务窗口: 新窗口默认最大化打开。
  • 单窗口模式: 最大化应用会自动最小化其他多任务窗口。
  • 自动结束画中画模式: 返回到应用窗口会自动结束 PiP 会话。
  • 跳过终止提示: 当应用退出时,窗口立即关闭,而不是显示终止摘要屏幕。
  • 终止后自动重启应用: 如果与 跳过终止提示 一起启用,应用窗口会在终止时自动重新启动。
  • 底部窗口栏: 将窗口控制栏移到屏幕底部。
  • 重定向URL到LiveContainer: 当多任务应用打开支持的 URL schemes 时,让主 LiveContainer 处理它们(而不是子进程),以提高应用到应用 URL 跳转的可靠性。
  • 快捷栏宽度: 调整多任务快捷栏的宽度。

容器行为​

  • 多任务实例基于容器。
  • 如果您启动已在多任务中运行的容器,LiveContainer 会将现有窗口带到前台,而不是创建副本。
  • 如果您在多任务应用活动时尝试标准启动,LiveContainer 会在继续之前要求确认。

JIT 与多任务​

如果为应用启用了 带JIT启动,多任务启动将首先进行 JIT 获取,然后继续进入多任务窗口。

为了获得最佳稳定性,请使用支持PID附加的 JIT 启用器(例如,StikDebug)。
这是多任务 + JIT 的推荐方法。
