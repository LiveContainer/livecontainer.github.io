# 应用设置

长按 LiveContainer 中的应用卡片并点击 `Settings` 打开应用设置。

## 数据部分

- **包文件名**: 显示应用包的文件名。
- **备注**: 自定义此应用的备注文本。
- **模块文件夹**:
  - 私有应用: 可选择插件文件夹
  - 共享应用: 在此页面显示为只读
- **转为共享App**: 将应用包及其数据/插件移动到共享文件夹,用于多个 LiveContainer 使用。
- **转为私有App**: 将共享应用及其数据/插件移回私有文件夹。

## 容器部分

- **容器列表**: 打开每个容器来管理容器级别的设置。
- **选择未使用的容器**: 附加现有的未使用容器文件夹(仅对私有应用显示)。

详见[多个/外部容器](./containers-and-external-data.md)

## 启动/JIT 部分

- **带JIT启动**: 将此应用标记为启动时需要 JIT。
- **JIT启动脚本**(iOS 26+): 为此应用附加自定义 JIT 启动脚本文件。

## 隐私部分

- **Lock App**: 启动前需要身份验证。
- **Hide App**: 应用锁定后可用;在正常列表中隐藏应用。

## 语言部分

- 设置应用特定语言
- `自动` 使用系统/默认行为

## 修复部分

- **修复文件导入**
- **修复本地通知**
- **使用LiveContainer的包名**

## 方向部分
如果应用存在到分辨率相关问题,特别是横向游戏,请尝试启用此选项

- 方向锁定: 禁用 / 横向 / 纵向

## 运行时兼容性部分
可能可以帮助绕过一些App的注入检测。

- **从Dyld API隐藏LiveContainer**
- **不注入TweakLoader**
- **不加载TweakLoader**(禁用注入后显示)

## SDK 行为部分

- **强制指定SDK版本**: 使系统库行为遵循App的SDK版本。

## 旧版文件选择器部分

- **(旧方法)修复文件导入**: 旧的文件选择器修复。它强制文件选择器将选定的文件复制到应用的Inbox文件夹。您可以在应用容器 -> Inbox 中找到并清理复制的文件。请注意,这可能会导致崩溃,或者在选择文件夹时可能无法工作。

## 签名部分

- **强制重新签名**: 立即强制重新签名此应用。
- 如果应用当前标记为"不签名",此操作将提示并首先重新启用签名。

## 统计部分

- 上次启动
- 安装日期
