# 应用设置

安装应用后，长按 LiveContainer 中的应用图标即可打开「设置」，自定义该应用的行为及与 LiveContainer 的交互方式。

## 数据

### 共享应用

将应用转换为共享应用后，可以在多个 LiveContainer 实例间运行。如果想删除该应用，请先改回私有应用再卸载。

## 容器（Containers）

点击容器 ID 即可编辑。若想在两个容器中打开同一应用或启用多账号，需要先设置「新建数据文件夹」，然后通过长按应用图标来切换数据文件夹。

### 使用 JIT 启动

启用后，启动该应用时会自动等待 JIT。

### 锁定应用

启用锁定后，打开应用需要 Face ID 或密码。

**隐藏应用**：如需彻底隐藏，请在设置中开启严格隐藏模式。

**严格隐藏模式**：认证后开启，隐藏的应用将不再在界面显示。重启 LiveContainer 后，在主界面三击「x 个应用」并通过验证即可解锁。

### 签名器

自 3.3.55 起，该选项已移除，默认使用 `ZSign`。

### 语言

与系统设置类似，可以单独为该应用指定语言。

## 修复项（FIXES）

### 修复文件选择器 & 本地通知

若遇到以下问题，可尝试启用：

- 无法选择文件
- 文件选择器无响应
- 应用要求开启通知才能继续

### 修复黑屏

如果应用显示黑屏并在 10 秒后崩溃，可启用此选项。

:::note
适用于 LiveContainer 3.1.55+
:::

3.1.55 之后不再自动启用黑屏修复。开启方法：长按应用 → 设置 → 启用「Black Screen Fix」。

### 对 Dyld API 隐藏 LiveContainer

有助于绕过部分应用的注入检测。

### 不注入 TweakLoader

同样用于尝试绕过注入检测。

### 伪装 SDK 版本

强制系统框架以应用构建时的 SDK 版本运行。

### （旧版）修复文件选择器

通过强制把选中的文件复制到 Inbox，解决点击「打开」无响应的问题。导入的文件可在应用数据目录下的 Inbox 查看。

### Bypass AssertBarrierOnQueue

~~可能防止部分游戏崩溃，但可能导致不稳定。~~  
3.3.55 已移除此选项。

### 强制签名

若应用因 “Invalid Signature” 等错误无法启动，可尝试重新签名。如果仍失败，请刷新 JIT-Less 证书。
