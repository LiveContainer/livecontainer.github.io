# 文档国际化流程

LiveContainer 文档提供英文和简体中文两个版本。本文档说明当你修改 Markdown 或 UI 文案时，如何同步翻译。

## 1. 生成翻译源文件

每次新增或重命名文档后，运行 Docusaurus 脚本：

```bash
npm run write-translations
# 仅处理单一语言：
npm run write-translations -- --locale zh-CN
```

该命令会更新：

- `i18n/zh-CN/docusaurus-plugin-content-docs/current/**` —— 对应的 Markdown 翻译副本
- `i18n/zh-CN/code.json` —— 由 MDX JSX 或元数据生成的 UI 字符串
- `i18n/zh-CN/docusaurus-theme-classic/*.json` —— 导航栏、页脚、搜索等主题文案

请把这些文件与英文改动一起提交，方便翻译者查看差异。

## 2. 编辑翻译 Markdown

每个英文文档都有固定 `id`，翻译文件需要放在 `i18n/zh-CN/docusaurus-plugin-content-docs/current` 下的相对路径。例如：

- 英文：`docs/guides/jit-support.md`
- 中文：`i18n/zh-CN/docusaurus-plugin-content-docs/current/guides/jit-support.md`

复制 front matter 和结构，然后翻译正文。保持代码块、Admonition、HTML 等元素与原文一致，确保 MDX 正常渲染。

如果通过 Crowdin 协作，推送最新源文件后在 Crowdin 完成翻译，再把结果同步回此目录。

## 3. 翻译主题字符串

导航栏、页脚、搜索等文案位于 `i18n/zh-CN/docusaurus-theme-classic`。当你修改 `docusaurus.config.js`（如新增导航条目）时，记得运行 `write-translations` 并更新这些 JSON。

## 4. 验证构建

请同时测试中英文站点：

```bash
npm run start -- --locale zh-CN
npm run build -- --locale zh-CN
```

若只需查看中文站，可以带 `--locale zh-CN` 启动开发服务器。建议在 CI 中也增加 `npm run build -- --locale zh-CN` 的检查，避免遗漏翻译导致构建失败。

## 5. 通知翻译者

当 PR 改动英文内容时，请备注是否需要翻译。如果暂时无法翻译，可在对应的 `i18n/zh-CN` 文件里留下 TODO，或创建带 `needs-translation` 标签的 Issue。
