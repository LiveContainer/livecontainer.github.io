# Workflow della Documentazione i18n 

LiveContainer ha la documentazione in Inglese, Cinese Semplificato e Italiano. Questa guida mostra come mantenere le traduzioni sincronizzate quando modifichi le pagine in Markdown o stringhe UI.

## 1. Genera le fonti per la traduzione

Esegui l'helper integrato in Docusaurus quando aggiungi o rinomini i documenti:

```bash
npm run write-translations
# o per una sola lingua:
npm run write-translations -- --locale zh-CN
```

Questo comando aggiorna:

- `i18n/zh-CN/docusaurus-plugin-content-docs/current/**` – copie Markdown tradotte.
- `i18n/zh-CN/code.json` – stringhe UI provenienti da MDX JSX, metadati, ecc.
- `i18n/zh-CN/docusaurus-theme-classic/*.json` – etichette per navbar/footer/ricerca.

Esegui il commit di questi file a fianco delle modifiche in Inglese cosicché i traduttori vedano le differenze.

## 2. Modifica Markdown tradotti

Ogni documento in Inglese mantiene il suo `id` originale. Il file tradotto deve esistere nello stesso percorso relativo dentro a `i18n/zh-CN/docusaurus-plugin-content-docs/current`. Esempio:

- Inglese: `docs/guides/jit-support.md`
- Cinese: `i18n/zh-CN/docusaurus-plugin-content-docs/current/guides/jit-support.md`

Copia alla lettera l'intestazione e la struttura, poi traduci il corpo del testo. Mantieni i blocchi di codice, le avvertenze e l'HTML inline esattamente dove si trovano, in modo che MDX si visualizzi correttamente.

Se usi Crowdin, fai il push delle source aggiornate, lascia che i traduttori lavorino lì, poi fai il pull delle traduzioni in questa cartella.

## 3. Traduci "theme strings"

Le etichette per Navbar/footer/ricerca si trovano sotto a `i18n/zh-CN/docusaurus-theme-classic`. Esegui `write-translation` e aggiorna i file JSON quando modifichi `docusaurus.config.js` (es. aggiungi un nuovo oggetto alla navbar).

## 4. Valida build

Testa sempre tutte le lingue localmente:

```bash
npm run start -- --locale zh-CN
npm run build -- --locale zh-CN
```

Esegui il dev server con `--locale zh-CN` se vuoi controllare solo il sito Cinese. Per build production, aggiungi un CI job che esegue `npm run build -- --locale zh-CN` così le traduzioni mancanti falliscono velocemente.

## 5. Informa i traduttori

Quando apri una PR che cambia il contenuto Inglese, scrivi se le traduzioni sono richieste. Se non puoi tradurre immediatamente, lascia un commento TODO nel file `i18n/zh-CN` relativo o crea un'issue taggando `needs-translation`.
